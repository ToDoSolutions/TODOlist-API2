HTTP URI Description
GET /groups Vista de todos los grupos que hay en la API.
  • order: Ordena todos los grupos siguiendo el orden: nombre, descripción, fecha de creación, usuarios, número de tareas.
  • limit: Limita el número de los grupos que aparecerán tras una petición.
  • offset: Muestra todas los grupos a partir de un número determinado.
  • fieldsGroup: Enseña de todos los grupos solamente los campos que se le especifique entre los siguientes: nombre, descripción, fecha de creación, usuarios, número de tareas.
  • fieldsUser: Enseña de todos los usuarios solamente los campos que se le especifique entre los siguientes: nombre, apellido, email, avatar, biografía, ubicación, tareas completadas.
  • fieldsTask: Enseña de todos los usuarios solamente los campos de las tareas que se le especifique entre los siguientes: título, descripción, anotación, estado, fecha de comienzo, fecha de finalización, prioridad, dificultad y duración.
  • name: Muestra todos los grupos que coincidan con el nombre que se ponga.
  • description: Muestra todos los usuarios que coincidan con la descripción que se ponga.
  • numTasks: Muestra todos los grupos a partir de los siguientes criterios.
    1. >:Devuelve aquellos que superen el número indicado.
    2. <:Devuelve aquellos que sean inferiores al número indicado.
    3. =:Devuelve aquellos que sean iguales al número indicado.
    Se pueden combinar hasta dos operaciones de las anteriores dadas.
  • createdDate: Enseña todos los grupos a partir de los siguientes criterios.
    1. >:Devuelve aquellas que posteriores a la fecha indicada.
    2. <:Devuelve aquellas que sean anteriores a la fecha indicada.
    3. =:Devuelve aquellas que sean iguales a la fecha indicada.
GET /groups/{groupId} Muestra el grupo cuya id coincida con la solicitada.
  • fieldsGroup: Enseña de todos los grupos solamente los campos que se le especifique entre los siguientes: nombre, descripción, fecha de creación, usuarios, número de tareas.
  • fieldsUser: Enseña de todos los usuarios solamente los campos que se le especifique entre los siguientes: nombre, apellido, email, avatar, biografía, ubicación, tareas completadas.
  • fieldsTask: Enseña de todos los usuarios solamente los campos de las tareas que se le especifique entre los siguientes: título, descripción, anotación, estado, fecha de comienzo, fecha de finalización, prioridad, dificultad y duración.
    • POST /groups Almacena un nuevo grupo que se guardará en formato JSON junto con los usuarios ya añadidos. (No es necesario poner la id)
    POST /groups/{groupId}/user/{userId} Añade a un grupo del sistema un nuevo usuario específico.
    POST /groups/{groupId}/task/{taskId} Añade a un grupo del sistema una nueva tarea específica.
    PUT /groups Actualiza los atributos de un grupo específico. (Se debe incluir la id del usuario en JSON)
    DELETE /groups/{groupId} Elimina el grupo cuya id coincida con la solicitada.
    DELETE /groups/{groupId}/user/{userId} Borra un usuario indicado por su id para un grupo determinado.
    DELETE /groups/{groupId}/task/{taskId} Borra la tarea de todos los usuarios de la organización que tengan esa tarea.
